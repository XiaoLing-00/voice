# 前端开发工程师面试核心知识点

## JavaScript 基础

### 事件循环（Event Loop）
JS 是单线程语言，通过事件循环处理异步：
- **调用栈（Call Stack）**：执行同步代码
- **宏任务（MacroTask）**：setTimeout、setInterval、I/O、UI rendering
- **微任务（MicroTask）**：Promise.then、MutationObserver、queueMicrotask
- 执行顺序：同步代码 → 清空微任务队列 → 取一个宏任务 → 清空微任务 → 循环

### 原型与继承
- 每个对象有 `__proto__` 指向其原型，函数有 `prototype` 属性
- 原型链：对象查找属性时沿 `__proto__` 链向上查找，直到 null
- ES6 class 是语法糖，本质仍是原型继承

### 闭包
函数能够访问其词法作用域外的变量。常见用途：模拟私有变量、防抖节流、柯里化。
注意：不当使用闭包会造成内存泄漏（变量无法被 GC）。

### Promise 与异步
- Promise 三种状态：pending → fulfilled / rejected，状态不可逆
- Promise.all：全部成功才 resolve，有一个失败就 reject
- Promise.allSettled：等所有都完成（无论成败）
- async/await 是 Promise 的语法糖，await 暂停当前 async 函数执行

## 浏览器原理

### 渲染流程
HTML 解析 → DOM Tree → CSSOM Tree → Render Tree → Layout → Paint → Composite
- **重排（Reflow）**：改变元素尺寸、位置，代价高
- **重绘（Repaint）**：改变颜色等外观，不影响布局，代价低
- 减少重排：批量修改 DOM、使用 documentFragment、CSS transform 代替位置变化

### 浏览器缓存
- **强缓存**：Cache-Control: max-age、Expires，不发请求直接用缓存
- **协商缓存**：Last-Modified/If-Modified-Since、ETag/If-None-Match，向服务器验证是否过期
- 缓存策略：HTML 不缓存（每次获取最新），JS/CSS 加 hash 强缓存

## Vue / React

### Vue 响应式原理
Vue2 使用 Object.defineProperty 拦截 getter/setter，Vue3 改用 Proxy，支持监听数组下标和新增属性。

### Vue 生命周期
beforeCreate → created → beforeMount → mounted → beforeUpdate → updated → beforeUnmount → unmounted

### React Hooks
- **useState**：管理组件状态
- **useEffect**：处理副作用（数据请求、DOM 操作），依赖数组控制执行时机
- **useCallback / useMemo**：缓存函数和计算结果，避免不必要的重渲染
- **useRef**：保存不触发渲染的可变值或 DOM 引用

### 虚拟 DOM 与 Diff 算法
虚拟 DOM 是真实 DOM 的 JS 对象描述，框架在内存中对比新旧虚拟 DOM（Diff），最小化真实 DOM 操作。
React Fiber：可中断的渲染架构，将渲染任务拆分为小单元，支持优先级调度。

## 工程化

### Webpack 核心概念
- **Entry**：入口文件
- **Output**：打包输出
- **Loader**：转换非 JS 文件（如 babel-loader、css-loader）
- **Plugin**：扩展功能（如 HtmlWebpackPlugin、MiniCssExtractPlugin）
- **Code Splitting**：代码分割，按需加载，减少首屏体积

### 性能优化
- 图片懒加载、骨架屏、虚拟列表
- Tree Shaking：移除未使用代码（需 ES Module）
- CDN 加速静态资源
- Service Worker 缓存
- 首屏优化：SSR/预渲染、preload/prefetch

## 网络

### HTTP/HTTPS
- **HTTP/1.1**：持久连接、管道化，但有队头阻塞
- **HTTP/2**：多路复用、头部压缩、服务器推送
- **HTTPS**：TLS 加密，防止中间人攻击

### 跨域
- **CORS**：服务端设置 Access-Control-Allow-Origin
- **JSONP**：利用 script 标签不受同源限制（仅 GET）
- **代理**：开发时 webpack devServer proxy，生产时 Nginx 反向代理

### 常见状态码
- 200 OK、201 Created、204 No Content
- 301 永久重定向、302 临时重定向、304 未修改（协商缓存）
- 400 Bad Request、401 未授权、403 禁止、404 Not Found
- 500 服务器错误、502 Bad Gateway、503 Service Unavailable
